# ADR-0003: 포즈 마커 특수 토큰화 (`<pause:short>` / `<pause:long>`)

- **Status**: Accepted
- **Date**: 2026-05-22

## Context

CHA 포맷의 포즈 마커 `(.)`(짧은 정지, 약 0.3–0.5초)와 `(..)`(긴 정지, 약 0.6–1.5초)를 전처리 단계에서 어떻게 처리할지 결정해야 한다. 후보는:

- (a) 제거
- (b) 원본 그대로 `(.)`, `(..)` 보존
- (c) 자연어 변환 `(short pause)` / `(long pause)`
- (d) 특수 토큰 `<pause:short>` / `<pause:long>`

본 프로젝트의 목표는 "자연스러운 구어체 생성"이며, 포즈는 자연 발화의 핵심 요소다. 또한 출력은 OPIc 응시자가 실제로 외워서 발화할 스크립트로 활용된다.

## Decision

**`<pause:short>` / `<pause:long>` 특수 토큰으로 보존한다.**

## Consequences

### 긍정적

- **학습 신호로 작동** — 모델이 자연 발화 중 끊김 위치를 학습 가능
- **출력 가치** — OPIc 응시자가 스크립트를 외울 때 "여기서 살짝 쉬세요" 가이드로 작동
- 토크나이저 친화적 형식 (명확한 boundary, 일관된 패턴)
- UI 출력 시점에 `…` 등 시각 표기로 후처리 변환 가능

### 부정적

- 토크나이저가 해당 토큰을 분리 토큰으로 인식하도록 추가 처리 필요 가능성
- 모델이 포즈를 과도하게 생성할 위험 → 학습 시 빈도 모니터링 필요

## Alternatives Considered

| 안 | 채택 안 한 이유 |
|---|---|
| 제거 | 자연 발화의 핵심 신호를 잃음 |
| 원본 `(.)` `(..)` 유지 | 일반 괄호와 혼동, 토크나이저 처리 모호 |
| 자연어 `(short pause)` | 토큰 길이 ↑, 모델이 일반 텍스트로 오인 가능 |
